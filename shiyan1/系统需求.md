# 若依管理系统 - 软件需求文档

## 1. 系统概述

### 1.1 系统简介

若依管理系统是一款基于 SpringBoot + Vue 技术栈的企业级后台管理框架，提供用户管理、角色权限、菜单配置、部门组织等核心功能模块，支持多租户和细粒度权限控制。

### 1.2 系统版本

- 版本号：v3.9.2
- 采集时间：2026-06-26
- 采集来源：https://vue.ruoyi.vip/index

### 1.3 技术栈

| 分类 | 技术 |
|------|------|
| 后端框架 | SpringBoot 4.0.3 |
| 安全框架 | Spring Security 5.7.14 |
| 认证方式 | JWT |
| ORM框架 | MyBatis |
| 连接池 | Druid 1.2.28 |
| JSON处理 | Fastjson 2.0.61 |
| 前端框架 | Vue 2.x |
| 状态管理 | Vuex |
| UI组件库 | Element-ui |
| HTTP客户端 | Axios 0.30.3 |
| CSS预处理器 | Sass |
| 富文本编辑器 | Quill |

## 2. 使用角色

### 2.1 角色定义

| 角色名称 | 角色标识 | 权限级别 | 职责描述 |
|----------|----------|----------|----------|
| 超级管理员 | admin | 最高权限 | 拥有系统所有功能权限，可管理所有用户、角色、菜单、部门等 |
| 普通操作员 | 自定义 | 受限权限 | 根据分配的角色权限访问系统功能，执行日常业务操作 |

### 2.2 角色权限差异

| 功能模块 | 超级管理员 | 普通操作员 |
|----------|------------|------------|
| 用户管理 | 增删改查 | 仅查看（视权限配置） |
| 角色管理 | 增删改查 | 无权限 |
| 菜单管理 | 增删改查 | 无权限 |
| 部门管理 | 增删改查 | 仅查看（视权限配置） |
| 岗位管理 | 增删改查 | 无权限 |
| 字典管理 | 增删改查 | 无权限 |
| 参数设置 | 增删改查 | 无权限 |
| 通知公告 | 增删改查 | 仅查看 |
| 操作日志 | 查看 | 查看（视权限配置） |
| 登录日志 | 查看 | 无权限 |

## 3. 系统管理模块功能需求

### 3.1 用户管理

**页面路径：** `/system/user`

#### 3.1.1 页面字段说明

| 字段名称 | 字段类型 | 数据来源 | 字段说明 |
|----------|----------|----------|----------|
| 用户编号 | Long | 系统生成 | 用户唯一标识，主键 |
| 用户名称 | String | 用户输入 | 用户登录账号，唯一 |
| 用户昵称 | String | 用户输入 | 用户显示名称 |
| 所属部门 | String | 树形选择 | 用户归属的部门 |
| 手机号码 | String | 用户输入 | 用户联系电话 |
| 邮箱地址 | String | 用户输入 | 用户电子邮箱 |
| 角色列表 | Array | 多选框 | 用户关联的角色 |
| 岗位列表 | Array | 多选框 | 用户关联的岗位 |
| 状态 | Integer | 下拉选择 | 0-禁用，1-启用 |
| 创建时间 | DateTime | 系统生成 | 记录创建时间 |
| 更新时间 | DateTime | 系统更新 | 记录最后更新时间 |

#### 3.1.2 业务规则

**新增用户：**
- 用户名称必填，且在系统内唯一
- 用户昵称必填
- 初始密码由系统自动生成，规则可配置
- 至少选择一个角色
- 所属部门为必填项
- 状态默认为启用

**编辑用户：**
- 不可修改用户名称
- 可修改用户昵称、部门、手机号码、邮箱、角色、岗位、状态
- 修改密码时需验证旧密码
- 用户归属部门可修改为空

**删除用户：**
- 支持单条删除和批量删除
- 超级管理员不可删除
- 已禁用用户可删除
- 删除前需确认

**分页查询：**
- 支持按用户名称模糊查询
- 支持按用户昵称模糊查询
- 支持按部门筛选（树形选择）
- 支持按状态筛选
- 支持分页，默认每页10条
- 支持排序（创建时间倒序）

### 3.2 角色管理

**页面路径：** `/system/role`

#### 3.2.1 页面字段说明

| 字段名称 | 字段类型 | 数据来源 | 字段说明 |
|----------|----------|----------|----------|
| 角色编号 | Long | 系统生成 | 角色唯一标识，主键 |
| 角色名称 | String | 用户输入 | 角色显示名称，唯一 |
| 角色权限 | String | 用户输入 | 角色权限标识，唯一 |
| 角色描述 | String | 用户输入 | 角色功能描述 |
| 数据范围 | Integer | 下拉选择 | 1-全部数据，2-本部门及以下，3-本部门，4-仅本人，5-自定义 |
| 状态 | Integer | 下拉选择 | 0-禁用，1-启用 |
| 创建时间 | DateTime | 系统生成 | 记录创建时间 |
| 更新时间 | DateTime | 系统更新 | 记录最后更新时间 |

#### 3.2.2 业务规则

**新增角色：**
- 角色名称必填，且在系统内唯一
- 角色权限必填，且在系统内唯一
- 角色权限必须以字母开头，只能包含小写字母、数字、下划线
- 数据范围默认为全部数据
- 状态默认为启用

**编辑角色：**
- 不可修改角色权限标识
- 可修改角色名称、角色描述、数据范围、状态
- 禁用角色后，该角色下的用户权限失效

**删除角色：**
- 支持单条删除和批量删除
- 超级管理员角色不可删除
- 已分配用户的角色删除前需先解除用户关联
- 删除前需确认

**权限分配：**
- 支持为角色分配菜单权限
- 权限以树形结构展示
- 支持全选/取消全选
- 分配权限后需保存生效

**分页查询：**
- 支持按角色名称模糊查询
- 支持按角色权限模糊查询
- 支持按状态筛选
- 支持分页，默认每页10条

### 3.3 菜单管理

**页面路径：** `/system/menu`

#### 3.3.1 页面字段说明

| 字段名称 | 字段类型 | 数据来源 | 字段说明 |
|----------|----------|----------|----------|
| 菜单编号 | Long | 系统生成 | 菜单唯一标识，主键 |
| 菜单名称 | String | 用户输入 | 菜单显示名称 |
| 上级菜单 | Long | 树形选择 | 父级菜单编号，顶级菜单为0 |
| 菜单类型 | Integer | 下拉选择 | 1-目录，2-菜单，3-按钮 |
| 路由地址 | String | 用户输入 | Vue路由路径 |
| 路由名称 | String | 用户输入 | 路由组件名称 |
| 组件路径 | String | 用户输入 | 组件文件路径 |
| 权限标识 | String | 用户输入 | 按钮权限标识 |
| 菜单图标 | String | 图标选择 | 菜单显示图标 |
| 排序号 | Integer | 用户输入 | 同级菜单排序顺序 |
| 是否显示 | Integer | 下拉选择 | 0-隐藏，1-显示 |
| 始终显示 | Integer | 下拉选择 | 0-否，1-是（即使无子菜单） |
| 状态 | Integer | 下拉选择 | 0-禁用，1-启用 |
| 创建时间 | DateTime | 系统生成 | 记录创建时间 |

#### 3.3.2 业务规则

**新增菜单：**
- 菜单名称必填
- 菜单类型必填
- 根据菜单类型显示不同字段：
  - 目录类型：需填写上级菜单、菜单名称、排序号、是否显示
  - 菜单类型：需填写上级菜单、菜单名称、路由地址、路由名称、组件路径、图标、排序号、是否显示
  - 按钮类型：需填写上级菜单、菜单名称、权限标识、排序号
- 路由地址和路由名称需要校验唯一性
- 排序号默认为0

**编辑菜单：**
- 可修改所有字段（除菜单编号）
- 修改路由地址或组件路径时需确保组件存在

**删除菜单：**
- 支持单条删除和批量删除
- 有子菜单的菜单不可删除，需先删除子菜单
- 删除前需确认

**批量保存排序：**
- 支持同一级菜单批量调整排序
- 通过拖拽方式调整顺序
- 保存后立即生效

**分页查询：**
- 以树形结构展示菜单
- 支持按菜单名称模糊查询
- 支持按菜单类型筛选
- 支持按状态筛选

## 4. 权限控制规则

### 4.1 权限体系架构

```
用户 ← 多对多 → 角色 ← 多对多 → 菜单/按钮权限
        ↓                              ↓
   数据权限配置                  功能权限配置
```

### 4.2 功能权限控制

#### 4.2.1 菜单权限
- 用户根据关联角色获取可访问的菜单列表
- 菜单以树形结构展示，权限继承上级菜单
- 未授权菜单在侧边栏不可见

#### 4.2.2 按钮权限
- 按钮权限通过权限标识（permission）控制
- 使用自定义指令 `v-permission` 控制按钮显示/隐藏
- 未授权按钮在页面中不可见

#### 4.2.3 接口权限
- 使用 `@PreAuthorize` 注解控制接口访问
- 注解参数为权限标识字符串
- 未授权请求返回403错误

### 4.3 数据权限控制

#### 4.3.1 数据范围类型

| 数据范围 | 标识 | 说明 |
|----------|------|------|
| 全部数据 | 1 | 可查看系统所有数据 |
| 本部门及以下 | 2 | 可查看本部门及下属部门数据 |
| 本部门 | 3 | 可查看本部门数据 |
| 仅本人 | 4 | 仅可查看本人创建的数据 |
| 自定义 | 5 | 可查看指定部门数据 |

#### 4.3.2 数据权限实现
- 使用 MyBatis 拦截器动态拼接数据权限SQL
- 根据用户角色的数据范围配置过滤数据
- 支持多角色数据权限的交集处理

### 4.4 权限缓存策略

- 用户权限信息缓存在 Redis 中
- 权限变更后自动清除缓存
- 支持手动刷新权限缓存
- 缓存过期时间可配置

## 7. 安全规则

### 7.1 登录安全

- 登录失败次数限制，超过阈值锁定账户
- 密码错误锁定时间可配置
- 登录IP黑名单限制
- 验证码机制防止暴力破解

### 7.2 数据安全

- 用户密码采用BCrypt加密存储
- 操作日志记录敏感操作
- 数据脱敏处理（身份证、手机号等）
- 防SQL注入攻击

### 7.3 请求安全

- JWT Token认证机制
- Token过期自动刷新
- 防重复提交机制
- XSS攻击防护

### 7.4 文件安全

- 文件上传类型校验
- 文件大小限制
- 文件名特殊字符过滤
- 防盗链功能
